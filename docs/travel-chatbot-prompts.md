# Travel Helper Prompt Contract

## Production System Prompt

```text
You are Travel Helper.
Answer travel questions clearly and practically.
If the user asks something unrelated to travel, politely say you only help with travel planning.
Return only JSON:
{"reply":"answer for the user","intent":{"hasDestinationIntent":false,"destination":""}}
```

## API Contract

The chat API returns only user-facing reply text, destination intent, and Worker-generated booking actions.

```json
{
  "reply": "The Maldives is a strong choice for quiet beaches and clear water.",
  "intent": {
    "hasDestinationIntent": true,
    "destination": "Maldives"
  },
  "actions": []
}
```

The Worker ignores model-provided actions and creates booking actions itself when the intent contains a destination.

## Backend Behavior

- The Worker calls only the main chat model for valid chat requests.
- There is no separate classifier model, keyword filter, planning state, readiness, budget extraction, generated suggestions, recommendation fallback, or hardcoded destination list.
- If the model returns malformed JSON, the Worker uses a safe travel-helper fallback reply.
- Destination actions are generated by the Worker, not trusted from the model.

## Action Contract

When a destination is selected, the Worker returns exactly two actions:

```json
[
  {
    "id": "book_flight",
    "label": "Book flight to Maldives",
    "url": "https://www.google.com/travel/flights/search?q=Flights%20to%20Maldives"
  },
  {
    "id": "book_hotel",
    "label": "Book hotel in Maldives",
    "url": "https://www.google.com/travel/hotels?q=Hotels%20in%20Maldives&destination=Maldives"
  }
]
```
